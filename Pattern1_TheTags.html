Knitting patterns have existed for millenia. The Library of Alexandria contained several created by Archimedes including his original pattern for woollen bathing trunks later revised after they fell off during his more famous Eureka moment.

Text, Lists, Tables and Forms

Text

<a> Identifies a hyperlink

<abbr> Identifies an abbreviation like Sunset Blvd. or an acronym like CSS

<acronym> Not supported in HTML5

<address> Identifies contact information for the author/owner of a document/article. For example,

Santa Claus
North Pole
The Arctic Circle

<b> Identifies bold text as a last resort if <em>, <strong> and <mark> have the night off

<basefont> Not supported in HTML5

<bdi> Isolates a part of text that might be formatted in a different direction from other text outside it

<bdo> Overrides the current text direction

<big> Not supported in HTML5

<blockquote> Identifies a section that is quoted from another source

<center> Not supported in HTML5

<cite> Identifies the title of a work. Distrust That Particular Flavor by William Gibson

<code> Identifies a piece of computer code in the text.Often used with <pre> Console.WriteLine("Hello");

<del> Identifies a text that has been deleted from a document, the time it was deleted and why. Compare with <ins>

<details>New Identifies additional details that the user can view or hide.

Note that this element is not appropriate for footnotes

<dfn> Identifies a term being defined in the text. For example, In astronomy, a syzygy is a straight line configuration of three celestial bodies in a gravitational system

<dir> Not supported in HTML5

<em> Identifies emphasized text

<figcaption>New Identifies a caption for a <figure> element

<figure>New Identifies a photo and supporting content in a document. For example

<font> Not supported in HTML5

<h1> Identifies a level 1 header

<h2> Identifies a level 2 header

<h3> Identifies a level 3 header

<h4> Identifies a level 4 header

<h5> Identifies a level 5 header
<h6> Identifies a level 6 header

<hgroup>New Groups heading (<h1> to <h6>) elements

A grouped level 1 header (title)

A grouped level 2 header (subtitle)

<hr> Change Identifies a thematic change in the content rather than just a horizontal line as in HTML4-. Thus the <hr> on the next line is semantically incorrect.

<i> Change Identifies a part of text in an alternate voice or mood rather than just a phrase to be italicized.

<img> Identifies an image. Contrast with <figure>

<ins> Identifies a text that has been inserted into a document, when and why

<kbd> Identifies keyboard input. For example, press Alt+F4 to close this browser

<mark>New Identifies marked/highlighted text

<p> Identifies a paragraph

<progress>New Represents the progress of a task. For example, 50%

<q> Identifies a short quotation, often inline. Use <blockquote> to mark up long quotations.

<ruby>New Identifies a ruby annotation (for East Asian typography)
<rp>New Identifies what to show in browsers that do not support ruby annotations
<rt>New Identifies an explanation/pronunciation of characters (for East Asian typography)

<s> Change Identifies text that is no longer correct rather than striketrhough text

<samp> Identifies sample output from a computer program

<small> Identifies smaller text such as copyright text

<strike> Not supported in HTML5

<strong> Change Identifies important text rather than text to be emphasized strongly

<sub> Identifies subscripted text

<summary>New Identifies a visible, clickable heading for a <details> element.

Click here for more on the summary element

The <summary> element should be the first child element of the <details> element.

<sup> Identifies superscripted text

<time>New Identifies a date/time. For example, January 1 2012

<tt> Not supported in HTML5

<u> Change Identifies text that should be stylistically different from normal text. Four exomple, a span of text with an unarticulated, though explicitly rendered, non-textual annotation, such as labeling the text as being a proper name in Chinese text (a Chinese proper name mark), or labeling the text as being misspelt

<var> Identifies a variable. For example, x2+y=z

<wbr>New Identifies a possible line-break. For example, Supercalifragilisticexpialidocious

Lists

  • <ul> Identifies an unordered (bulleted) list
  • <li> Identifies a list item
  • The ‘compact’ and ‘type’ attributes of <ul> are not supported in HTML5
  1. <ol> Identifies an ordered list
  2. <li> Identifies a list item
  3. Numbering starts with 1 by default. Use the ol ‘start’ attribute to change the default
  4. Use ‘reversed’ to have the list count down rather than up
  5. The ‘compact’ and ‘type’ attributes of <ol> are not supported in HTML5
<dl>
Change Identifies an association list of name-value pairs rather than a list of terms and definitions
<dt>
Identifies the name (the term) in an association list
<dd>
Identifies the value (a description of the term) in an association list

New The <menu> tag defines a list/menu of commands. It is used for context menus, toolbars and for listing form controls and commands.

    Form Elements

    <fieldset> group related form items. Visually this is done by drawing a box around them. <legend> provides a caption for the fieldset
    <button> Identifies a clickable button

    Use the <input> element’s list attribute to bind it together with a <datalist> element.

    <keygen>New Identifies a key-pair generator field (for forms)

    When the control’s form is submitted, the private key is stored in the local keystore, and the public key is packaged and sent to the server.
